# Структура проекта

1. cmd - основные приложения проекта.
2. internal - внутренний код приложения и библиотек. Это код, который не должен использоваться в других приложениях и библиотеках.ы
3. pkg - код библиотек, пригодных для использоваения в сторонних приложениях.
4. configs - шаблоны файлов конфигураций и файлы настроек по-умолчанию.
5. migrations - миграции БД.

# Структура модуля

1. handler - http обработчик запроса.
2. payload - описание структур передаваемых и возвращаемых данных.
3. repository - репозиторий доступа к БД.
4. errors - список errors для модуля.
5. service - дополнительный сервис с бизнес логикой.
