# Структура проекта

1. cmd - основные приложения проекта.
2. internal - внутренний код приложения и библиотек. Это код, который не должен использоваться в других приложениях и библиотеках.
3. packages - код библиотек, пригодных для использоваения в сторонних приложениях.
4. configs - шаблоны файлов конфигураций и файлы настроек по-умолчанию.
5. migrations - миграции БД.

# Структура модуля

1. handler (controller) - http обработчик запроса.
2. payload (dto) - описание структур передаваемых и возвращаемых данных.
3. errors (filter) - список errors для модуля.
4. service - дополнительный сервис с бизнес логикой.
5. repository - репозиторий доступа к БД.
